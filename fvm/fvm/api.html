enum axis

Mesh axes.

The axis identifies a Cartesian coordinate axis on the mesh.

Values:

enumerator x_axis

X-coordinate axis.

enumerator y_axis

Y-coordinate axis.

enumerator z_axis

Z-coordinate axis.

enum boundary

Boundary.

Identifies the low or high boundary along a particular axis.

Values:

enumerator low

Low axis boundary.

enumerator high

High axis boundary.

enum boundary_type

Boundary type.

The supported boundary types are derived from Leveque’s Finite Volume Methods for Hyperbolc Problems. All boundary conditions are implemented using the ghost-cell approach outlined in the text.

Values:

enumerator flow

Zero-order extrapolation from the interior solution.

enumerator reflecting

Cauchy problem solution for solid wall boundary.

enumerator periodic

Substitution with opposite interior solution.

enum domain

Mesh domains.

The domain identifies the supported iteration spaces on the mesh.

Values:

enumerator quantities

This domain includes the locations of the unknowns of the problem state.

enumerator predictor

This domain includes all mesh locations where it is possible to compute a 2nd-order slope approximation.

enumerator corrector

This domain includes all mesh locations where it is possible to compute a flux across an interface from reconstructed predictor quantities.

enumerator all

This domain includes all local cells, including halo and boundary cells.

enumerator global

This domain includes all global cells.

template<axis A, domain DM = quantities, bool R = false>
inline auto cells() const

Return a range over the given axis and domain.

The range can be used to iterate over the cells in the given domain, e.g.: 

for(auto k: m.cells<mesh::z_axis, mesh::quantities>()) {
  for(auto j: m.cells<mesh::y_axis, mesh::quantities>()) {
    for(auto i: m.cells<mesh::x_axis, mesh::quantities>()) {
      u[k][j][i] = 1.0;
    } // for
  } // for
} // for

Template Parameters:

  • A – The mesh axis.

  • DM – The mesh domain (excluding global).
